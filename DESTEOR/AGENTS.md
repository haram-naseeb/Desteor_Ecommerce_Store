1. Project Overview





DESTEOR is a premium artificial jewellery e-commerce platform.



It is intended to become a real business, not just a portfolio project.



The codebase should always prioritize scalability, maintainability, clean architecture, and production readiness.

5. Architecture Rules





Headless architecture.



Client and server remain completely separate.



Backend layering:





Controller



Service



Repository



Prisma



Business logic belongs only in Services.



Controllers stay thin.

6. Coding Standards





Reuse components.



No duplicated code.



Use the @ alias.



Build reusable UI.



Keep everything responsive.



Accessibility matters.



Follow clean code principles.

7. Development Workflow





Implement one feature at a time.



Explain architecture before major changes.



Output only new or modified files.



Never regenerate the entire project.



Verify imports and compilation before finishing.



Keep documentation updated when architecture changes.
